# ARTIFACT INDEX MODEL

## Purpose

Provide one lightweight catalog of the repository-side worker artifacts and the linked real-run manifests.

## Artifact index outputs

Main files:

- `C:\Tools\SOLIDWORKS-MCP\artifacts\indexes\worker-artifact-index.json`
- `C:\Tools\SOLIDWORKS-MCP\artifacts\indexes\worker-artifact-index-summary.txt`

Generated by:

```powershell
pwsh -File C:\Tools\SOLIDWORKS-MCP\scripts\update-worker-artifact-index.ps1
```

## Top-level structure

- `schemaVersion`
- `generatedAt`
- `projectRoot`
- `validationRoot`
- `roots`
- `latest`
- `preferred`
- `counts`
- `recent`

## `roots`

Records the current directories used for:

- cycle artifacts
- evidence bundles
- regression manifests
- handoff packs
- index files

## `latest`

Tracks the most useful single entries, including:

- latest cycle
- latest passing cycle
- latest standard cycle
- latest full-evidence cycle
- latest evidence bundle
- latest passing evidence bundle
- latest standard evidence bundle
- latest full-evidence bundle
- latest regression run
- latest handoff pack

## `preferred`

Provides the current recommended references for routine use:

- `routineCycle`
- `fullEvidenceBundle`
- `handoffSourceBundle`

This keeps another developer from having to infer the best candidate manually.

## `counts`

Quick inventory numbers for:

- cycles
- evidence bundles
- regression runs
- handoff packs

## `recent`

Bounded recent lists for:

- cycle entries
- evidence bundle entries
- regression run entries
- handoff pack entries

The current script intentionally caps those lists rather than mirroring the whole artifact store forever.

## Compatibility note

Older manifests may not contain explicit `bundleMode`.

The indexer infers it from `includeCadFiles` when possible:

- `false` -> `standard_minimal`
- `true` -> `full_with_cad_files`

That keeps older cycle and evidence entries visible without rewriting historical manifests.
